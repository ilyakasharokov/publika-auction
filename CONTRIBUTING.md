# Contributing to Publika Auction Bot

Спасибо за интерес к проекту! Вот руководство по контрибьютингу.

## 📋 Процесс

1. **Форкните репозиторий**
   ```bash
   git clone https://github.com/ilyakasharokov/publika-auction.git
   cd publika-auction
   ```

2. **Создайте ветку для своей функции**
   ```bash
   git checkout -b feature/your-feature-name
   # или
   git checkout -b bugfix/your-bug-fix
   ```

3. **Внесите изменения**
   - Следуйте стилю кода проекта
   - Добавьте тесты для новых функций
   - Обновите документацию при необходимости

4. **Протестируйте свои изменения**
   ```bash
   make test
   make lint
   ```

5. **Сделайте коммит**
   ```bash
   git commit -am "feat: Add new feature"
   # или
   git commit -am "fix: Fix bug in component"
   ```
   
   Используйте conventional commits:
   - `feat:` - новая функция
   - `fix:` - исправление ошибки
   - `docs:` - документация
   - `style:` - форматирование кода
   - `refactor:` - рефакторинг
   - `test:` - добавление тестов
   - `chore:` - обновление зависимостей

6. **Пушьте в вашу ветку**
   ```bash
   git push origin feature/your-feature-name
   ```

7. **Откройте Pull Request**
   - Описание должно быть понятным
   - Упомяните связанные issues
   - Приложите скриншоты для UI изменений

## 📝 Стиль кода

- Используйте `gofmt` для форматирования
- Следуйте [Effective Go](https://golang.org/doc/effective_go)
- Писывайте понятные комментарии для сложного кода
- Используйте значимые имена переменных и функций

## 🧪 Тестирование

- Все новые функции должны иметь тесты
- Покрытие должно быть не ниже 80%
- Используйте table-driven tests где применимо

## 📚 Документация

- Обновляйте README.md при добавлении новых функций
- Добавляйте примеры использования
- Документируйте публичные функции

## 🐛 Репортинг ошибок

При нахождении ошибки:
1. Проверьте, не существует ли уже такой issue
2. Откройте новый issue с:
   - Описанием проблемы
   - Шагами воспроизведения
   - Ожидаемым результатом
   - Фактическим результатом
   - Информацией о вашей среде (Go версия, OS и т.д.)

## 💡 Предложения по функциям

1. Проверьте существующие issues
2. Откройте issue с тегом `enhancement`
3. Опишите использование и преимущества функции
4. Обсудите реализацию

## 📞 Вопросы?

Откройте discussion или issue, и мы вам поможем!
